# SC Mod Launcher Releases

## 1.0.3

### Что нового

- Фильтры чертежей в `Майнинг и крафт` и `Квесты и рецепты` стали вложенными: категория, подкатегория, конкретное семейство.

- Корабельные компоненты в общем фильтре явно ограничены `A`-грейдом.
- Исправлена группировка `MacFlex "Rucksack" Backpack`: теперь он входит в `MacFlex set`.

- Корневая папка `reports` больше не нужна игроку и не создаётся как пользовательский артефакт.

## 1.0.0

### Что нового

- Создан первый релиз общего `SC Mod Launcher`.
- Добавлены два manifest-модуля: `Квесты и рецепты` и `Майнинг и крафт`.
- Добавлен общий core для загрузки модулей, чтения `global.ini`, ремонта битых `EM`-тегов, backup/report, dry-run/apply pipeline и проверки конфликтов.
- Модуль `Майнинг и крафт` фильтрует методы добычи `[К]`, `[Т]`, `[М]` в SCMDB-подсказках планет по выбранным галкам и добавляет состав предметов в описания.
- Опции модуля `Квесты и рецепты` перестроены под категории чертежей: корабельные компоненты, корабельные орудия, броня/одежда и FPS-оружие. Метка `[Ч]` ставится только для контрактов, где есть выбранные категории; `[А]` и `[С]` остаются всегда включёнными.
- Модуль `Квесты и рецепты` строит quest operations, фильтрует blueprint-блоки по выбранным категориям и сохраняет `[А]`/`[С]` как always-on маркеры.
- Добавлен первый compiled WPF launcher shell: тёмный sci-fi/космопиратский стиль, рабочие экраны overview/modules/updates, подпись `johnniewalker89`, и безопасные кнопки проверки/LIVE apply через существующий backend.
- Подключена первая проверка GitHub Releases для лаунчера: экран обновлений запрашивает GitHub API, ищет именно asset `SC_Mod_Launcher_*.zip`, показывает текущую/последнюю версию и не путает лаунчер с релизами других модов.
- Добавлены скачивание update ZIP, проверка SHA-256 и helper самообновления лаунчера с backup текущих файлов перед заменой.
- Релизный ZIP теперь содержит `update-manifest.json`: updater проверяет файлы пакета, зеркально обновляет управляемые зоны, удаляет устаревшие файлы старых версий и сохраняет `backups`, cache модулей и `updates\backups`.
- Пользовательский release ZIP содержит только runtime-файлы и update-helper; dev scripts `Build-*`, `Test-*`, `src/bin`, `dist`, `reports`, `.pdb` и старые удалённые модули вычищаются updater-ом.
- GUI получил отдельную кнопку `Применить в LIVE`: она работает как явное действие с preflight источников, backup `global.ini` и JSON-отчётом.
- GUI получил вкладку `Backup`: восстановление последнего или выбранного `global.ini` backup предварительно сохраняет текущий LIVE.
- Во вкладке `Backup` можно удалить выбранный backup в корзину Windows без ручного открытия папки.
- Старый WinForms launcher entrypoint и batch-запускатели удалены из пользовательской поставки; основным запуском стал `SC_Mod_Launcher.exe`, а `SC_Mod_Launcher.ps1` работает как backend/CLI.
- WPF-визуал усилен до command-deck scaffold: вместо левой древовидной навигации используется нижняя корабельная панель с аппаратными кнопками, активная секция подсвечивается, центральный main display получил HUD/radar-анимацию, а лог вынесен в отдельный shipboard panel.
- Панель корабельного техника уточнена после тестовых скриншотов: фон виднее, путь читается резче, лишняя крутилка в заголовке убрана, декоративный радар оставлен без подписей, а основные кнопки переведены на русский.
- Добавлен WPF smoke-тест: он собирает/запускает лаунчер, проверяет ключевые русские кнопки и разделы через UI Automation, затем закрывает процесс.
- Релизный ZIP теперь перед упаковкой запускает scaffold-тест backend, сборку WPF и WPF smoke-тест.

### Что умеет сейчас

- Показывает модули и options с галками.
- Проверяет путь к `StarCitizen\LIVE`.
- Создаёт dry-run JSON report без записи в игру.
- Общий core умеет применять patch-set к целевому `global.ini` с backup; GUI даёт отдельные действия для проверки и LIVE apply.
- Модуль `Квесты и рецепты` умеет добавлять/фильтровать SCMDB-блоки квестовых наград и обновлять `[Ч]` по выбранным категориям.
- При выборе только корабельной добычи dry-run показывает, какие planet description строки будут очищены от наземных/ручных фрагментов.
- Dry-run summary теперь показывает выбранные галки, методы добычи и количество изменяемых описаний без лишней повторяющейся строки про LIVE.
- WPF-лаунчер запускается через `SC_Mod_Launcher.exe` после сборки `tools\Build-WpfLauncher.ps1`.
- Кнопка `Проверить обновления` проверяет GitHub Releases и останавливается красным контуром при недоступном источнике.
- После найденного релиза кнопка `Обновить` загружает ZIP в `updates\downloads`; установка запускается только если SHA-256 совпал с опубликованным хэшем.
- `Применить в LIVE` записывает настоящий `global.ini` как отдельное явное действие; перед записью создаётся backup.
- Перед отдачей сборки можно прогнать `tools\Build-WpfLauncher.ps1 -RunSmokeTest`.
